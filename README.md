# FileUniquefy 微信文件去重工具

扫描微信文件目录下所有形如 `YYYY-MM` 的子文件夹，自动识别重复文件，将重复文件移动到上层的「重复」文件夹中，保持原始目录结构。

## 功能描述

- **自动发现**：启动后自动扫描系统中的微信文件目录，支持新旧两版微信路径
- **概览分析**：选定目录后立即展示各月份的文件数和 MD5 精确重复数
- **MD5 检测**：通过计算文件 MD5 哈希值精确判断文件是否重复，大小相同才进入 MD5 比较
- **只处理一层**：仅扫描 `YYYY-MM` 下的直接文件，不递归进入子文件夹
- **智能保留**：MD5 相同的重复组中，优先保留无 `(N)` 后缀的原始文件；同类情况下再按修改时间保留最旧的
- **结构保持**：移动后的文件保持原始的目录层级结构
- **四阶段流程**：选目录 → 概览确认 → 移动计划 → 执行
- **返回主界面**：执行完成后可返回重新选择其他目录

## 支持的微信目录格式

| 版本 | 路径格式 |
|------|---------|
| 新版微信 | `xwechat_files/<账号>/msg/file/` |
| 旧版微信 | `WeChat Files/<账号>/FileStorage/File/` |

脚本会自动扫描所有盘符（Windows）或用户目录（macOS/Linux）下的常见位置，包括自定义的 Documents 路径。

## 安全防护

- 禁止对根目录、用户主目录、系统关键目录操作
- 路径层级过浅时要求确认
- 非微信路径要求额外确认
- 待移动文件数量超过 500 时额外警告

## 保留规则说明

微信在同一文件夹中存入重复文件时，会将副本命名为 `文件名(1).ext`、`文件名(2).ext` 等。去重时：

1. **仅对 MD5 完全相同的文件进行处理**，不同内容的文件不受影响
2. **优先保留无 `(N)` 后缀的文件**（原始命名），移动带 `(1)`、`(2)` 等后缀的副本
3. 若同一 MD5 组内文件均无后缀或均有后缀，则保留修改时间最早的

## 目录结构示例

```
WeChat Files/xxx/FileStorage/File/
├── 2025-11/
│   ├── report.pdf
│   └── photo.jpg
├── 2025-12/
│   ├── a.pdf          ← 保留（原始文件，无后缀）
│   ├── a(1).pdf       ← MD5 相同，移动到「重复」
│   └── subdir/        ← 子文件夹，不处理
└── 重复/                ← 自动创建
    └── 2025-12/
        └── a(1).pdf   ← 移动到这里
```

## 使用方法

```bash
# 自动扫描微信目录，交互式选择
python FileUniquefy.py

# 通过命令行参数直接指定目录
python FileUniquefy.py "D:\document\WeChat Files\bachopin\FileStorage\File"
```
